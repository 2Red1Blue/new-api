# 模型选择

可用模型会受到账号权限、令牌分组、模型限制和当前服务状态影响。最可靠的方式是用自己的 Key 查询模型列表。

## 查看模型列表

```bash
curl https://ai.laiber.cloud/v1/models \
  -H "Authorization: Bearer $LAIBER_API_KEY"
```

如果某个模型没有出现在返回结果中，当前 Key 通常无法调用它。

## 选择建议

- 日常对话优先使用轻量模型，延迟低、成本低。
- 代码生成、复杂推理、长上下文分析使用更高能力模型。
- 图片、音频、Embedding、Rerank 等任务需要选择对应类型模型。
- 生产服务中尽量固定模型名称，避免客户端随意切换导致费用波动。

## 常见问题

### 模型不存在

先调用 `/v1/models` 确认模型是否可见，再检查 API Key 是否限制模型范围。

### 上游返回模型不支持

可能是该模型当前不可用，或者你的 Key 没有对应权限。先换一个模型测试；如果多个模型都不可用，再联系站点支持。
